# 《过拟合恋人》AI 协作指引

## 1. 使用目标

本文档用于指导在 Cursor、Trae 或其他编程助手中协作开发《过拟合恋人》。

所有 AI 协作任务都必须服从 `ta说_PRD_最终版.md`，并以正式开发要求为准。

## 2. 核心原则

1. 不允许 AI 用静态页、假数据、预设结果代替正式功能。
2. 不允许 AI 把“以后再补”写成当前需求的默认前提。
3. 剧情推进由规则系统控制。
4. 模型只负责受约束的文本生成。
5. 输出结构必须可验证、可渲染、可复用。

## 3. 推荐提问结构

每次给 AI 任务时，至少包含：

1. 背景
2. 范围
3. 限制
4. 输入输出结构
5. 正式验收标准

## 4. 固定约束模板

```text
你是《过拟合恋人》项目的协作工程师。请严格基于最终版 PRD、现有目录结构和共享类型工作。
不要以静态页、假数据、预设结果或临时替代逻辑代替正式功能。
不要让 LLM 决定剧情推进。
页面组件不要直接承担模型请求。
如果共享类型、输出 schema 或接口契约需要变更，请先说明影响范围。
```

## 5. Review 清单

AI 改完后必须检查：

1. 是否偏离 PRD
2. 是否用代偿实现冒充正式功能
3. 是否把规则职责交给模型
4. 是否破坏共享类型
5. 是否同步更新了相关文档
