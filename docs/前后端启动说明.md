# 《ta说》前后端启动说明

## 1. 当前项目现状

当前仓库已经可以直接启动前端页面，但**还没有独立的后端服务目录**。

目前的运行方式是：

- 前端：`Vite + React + TypeScript`
- 本地接口：由 `vite.config.ts` 中的开发插件临时提供
- 已内置的本地 mock 接口：
  - `POST /api/love-translate`
  - `POST /api/share-line`

也就是说，**现在本地开发时只需要启动一个前端服务**，就可以同时调通页面和这两个接口。

## 2. 环境要求

建议环境：

- `Node.js 18+`
- `npm 9+`

如果你还没装 Node，可以先执行：

```bash
node -v
npm -v
```

能正常输出版本号即可。

## 3. 第一次启动

在项目根目录 `D:\ta-say` 打开终端，执行：

```bash
npm install
```

安装完成后再启动开发环境。

## 4. 前端怎么打开

### 开发模式

在项目根目录执行：

```bash
npm run dev
```

启动成功后，终端会看到类似：

```text
Local:   http://localhost:5173/
```

然后浏览器打开：

```text
http://localhost:5173/
```

当前可访问的页面主要有：

- 首页：`http://localhost:5173/`
- 翻译官：`http://localhost:5173/translator`
- 分享卡：`http://localhost:5173/share`

### 生产预览

如果你想先打包再本地预览：

```bash
npm run build
npm run preview
```

## 5. 后端怎么打开

### 当前答案

**现在没有单独后端进程需要启动。**

当前“后端接口”由 `vite.config.ts` 里的开发插件直接挂在 Vite 服务上，所以：

- 执行 `npm run dev` 后
- 前端页面和本地 mock 接口会一起启动

也就是说，现阶段你看到的“后端”其实是：

- 前端 dev server
- 加上内置的 mock API

### 当前已经可用的本地接口

启动 `npm run dev` 后，可直接访问这些接口：

- `POST /api/love-translate`
- `POST /api/share-line`

对应逻辑位置在：

- [vite.config.ts](file:///d:/ta-say/vite.config.ts)

## 6. 怎么判断前后端都正常

启动 `npm run dev` 后，可以按下面方式检查：

1. 浏览器打开 `http://localhost:5173/`
2. 进入翻译官页 `http://localhost:5173/translator`
3. 点击“生成翻译报告”
4. 如果页面能成功出现翻译结果，说明：
   - 前端页面正常
   - 本地 mock 接口正常
5. 再进入 `http://localhost:5173/share`
6. 如果分享卡能显示，说明前后链路基本正常

## 8. 常见问题

### 1. 为什么我没看到后端目录？

因为当前版本还没接真实独立后端，接口先用 `vite.config.ts` 里的本地 mock 顶上，方便前端独立开发和演示。

### 2. 为什么启动前端后接口也能用？

因为 Vite 开发服务器启动时，会同时注册本地接口中间件。

### 3. 后面如果接真实后端怎么办？

后续如果项目要接真实接口，建议这样改：

1. 新建独立的 `api/` 或服务端工程目录
2. 把 `vite.config.ts` 里的 mock 逻辑迁走
3. 前端继续通过 `/api/...` 调用
4. 开发环境改为：
   - 一个终端启动前端
   - 一个终端启动真实后端

到那时，启动方式会变成：

```bash
# 终端 1
npm run dev

# 终端 2
后端启动命令
```

## 9. 一句话总结

当前项目阶段：

- **前端要单独启动**
- **后端不用单独启动**
- **运行 `npm run dev` 就等于同时打开了前端 + 本地 mock 接口**
