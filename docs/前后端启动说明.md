# 《过拟合恋人》前后端启动说明

## 1. 当前要求

本项目按正式开发版要求推进：

- 聊天正式回复能力必须可用
- 恋爱翻译官正式生成能力必须可用
- 分享卡正式数据链路必须可用
- 语音输入正式能力必须可用

不以开发期假接口、静态结果或预设代偿作为交付方式。

## 2. 环境要求

- `Node.js 18+`
- `npm 9+`

## 3. 安装依赖

```bash
npm install
```

## 4. 配置环境变量

```bash
cp .env.example .env
```

在 `.env` 中填写真实值：

```bash
VITE_LLM_API_KEY=your_api_key
VITE_LLM_BASE_URL=https://api.openai.com/v1
VITE_LLM_MODEL=gpt-4o-mini
```

`.env.example` 仅是模板，`.env` 才是实际配置。

## 5. 启动开发环境

```bash
npm run dev
```

默认地址：

```text
http://localhost:5173/
```

## 6. 正式接口要求

系统应按 PRD 对齐以下接口：

- `POST /api/anti-translate`
- `POST /api/ta-reply`
- `POST /api/love-translate`
- `POST /api/share-line`

如果当前仓库仍残留开发期本地接口，它们不能作为正式完成标准。

## 7. 常用命令

```bash
npm install
npm run dev
npm run lint
npm run test
npm run build
npm run preview
```

## 8. 联调检查

### 聊天

1. 聊天页可发送
2. 第三次发送触发污染
3. `TA` 回复来自正式生成链路

### 翻译官

1. 输入聊天内容后能生成正式结构化结果
2. 输出字段完整

### 分享卡

1. 基于真实会话和翻译结果生成
2. 支持导出

### 语音输入

1. 可真实采集语音并转写
2. 若环境不支持，明确提示不可用，而不是以预设文本冒充完成
