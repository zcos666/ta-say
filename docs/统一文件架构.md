# 《过拟合恋人》统一文件架构

## 1. 目标

定义仓库边界，确保正式功能有清晰归属。

## 2. 顶层目录

```text
ta-say/
  README.md
  CHANGELOG.md
  docs/
  src/
  tests/
  public/
  .trae/
```

## 3. `src/` 结构

```text
src/
  app/
  pages/
  components/
  features/
  services/
  config/
  types/
```

## 4. 约束

1. 页面组件不直接承担正式接口请求。
2. 正式能力必须落在对应 feature / service 中。
3. 不允许把静态占位实现混入正式链路。
4. 目录职责变更时必须同步文档。
