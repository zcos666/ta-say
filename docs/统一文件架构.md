# 《ta说》统一文件架构

## 1. 目标

本文件定义统一仓库骨架，目标是：

1. 让开发者 A 和 B 可以并行开发。
2. 让共享目录、归属目录和禁改目录足够明确。
3. 让后续 `Trae` 协作时有稳定的文件边界。


## 5. 共享目录改动规则

共享目录包括：

- `src/features/story`
- `src/services/api`
- `src/services/storage`
- `src/types`
- `src/config`
- `src/app/routes`
- `src/app/store`

规则：

1. 修改前先同步。
2. 不要在共享目录里写只属于某一人的页面逻辑。
3. 类型变更必须同时更新消费方文档。
4. 状态字段命名不允许出现两个临时版本。

## 6. 目录命名规则

1. 页面目录统一 `PascalCase`，例如 `ChatPage`。
2. 业务能力目录统一 `kebab-case`，例如 `save-load`。
3. 服务目录名使用能力词，不使用临时名称。
4. 共享类型放到 `src/types`，不要散落到页面目录。

## 7. 文件落地规则

推荐单文件职责：

- `index.tsx` 或 `Page.tsx`：页面入口
- `hooks.ts`：页面专属 hook
- `constants.ts`：页面静态文案或配置
- `types.ts`：仅页面内部类型，不能和共享类型重复
- `README.md`：该目录职责说明

## 8. 禁止事项

1. 不要把 API 请求直接写在页面组件里。
2. 不要把 localStorage 操作散落到多个页面。
3. 不要在 `TranslatorPage` 目录里改聊天主逻辑。
4. 不要在 `ChatPage` 目录里生成分享卡数据结构。

## 9. 变更原则

这个文件不是展示树状图而已，而是开发边界协议。后续如果目录需要变更，应先修改本文件，再调整实际目录。
