# 《过拟合恋人》版本规范

## 1. 目标

本文件用于统一项目阶段、提交版本和对外说明口径。

所有口径以 `ta说_PRD_最终版.md` 为准。

## 2. 当前版本定义

- 当前阶段名称：`正式开发版`
- 当前含义：按正式功能要求推进，实现完整产品链路，不接受实质功能代偿

## 3. 版本格式

```text
v主版本.次版本.修订号-阶段标签
```

示例：

- `v0.5.0-formal-dev`
- `v0.6.0-integration`
- `v0.7.0-release-candidate`
- `v1.0.0-launch`

## 4. 升级规则

### 升次版本

满足任一条件时升级次版本：

- 一个正式功能链路完成
- 一个共享契约稳定落地
- 一次重要集成完成

### 升修订号

满足任一条件时升级修订号：

- 修复 bug
- 调整文案
- 优化交互或样式
- 更新测试或文档

## 5. 版本纪律

以下情况不得视为版本阶段推进：

- 只补了静态页
- 只补了假数据
- 只补了展示层未补正式功能
- 只用预设内容代替正式输入或正式生成
