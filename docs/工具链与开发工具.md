# 《过拟合恋人》工具链与开发工具

## 1. 目标

统一项目开发工具、实现边界和协作约束，确保大家是在完成正式功能，而不是拼接代偿方案。

## 2. 推荐技术栈

- `Vite`
- `React`
- `TypeScript`
- `Zustand`
- `React Router`
- `html-to-image`

## 3. LLM 与接口原则

1. 剧情推进靠规则。
2. 文本细节靠 LLM。
3. 关键功能不得由假数据或静态结果代替。
4. 页面组件不直接写模型请求。

## 4. 浏览器能力

- `localStorage`：存档、通关状态、分享卡缓存
- 真实语音识别能力：语音输入
- `html-to-image`：导出分享卡

## 5. 测试重点

1. 反义污染触发是否正确
2. 读档三状态是否正确
3. 恋爱翻译官是否生成结构化结果
4. 分享卡是否依赖真实状态
5. 语音输入是否为真实能力而非代偿
