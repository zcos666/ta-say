# 《过拟合恋人》协作分工与目录归属

## 1. 总原则

本项目采用双人并行开发，但目标是完成同一个正式产品。

- 开发者 A 负责前 80% 的恋爱 Meta 恐怖游戏体验
- 开发者 B 负责后 20% 的恋爱翻译官、分享卡与语音输入
- 所有实现要求以 `ta说_PRD_最终版.md` 为准

## 2. 开发者 A 负责

- `src/pages/StartPage`
- `src/pages/SelectionPage`
- `src/pages/ChatPage`
- `src/pages/SpacePage`
- `src/pages/TruthPage`
- `src/pages/WakePage`
- `src/features/pollution`
- `src/features/reply`
- `src/features/drafts`
- `src/features/save-load`

## 3. 开发者 B 负责

- `src/pages/TranslatorPage`
- `src/pages/ShareCardPage`
- `src/features/translator`
- `src/features/share-card`
- `src/services/voice`

## 4. 共享区域

- `src/types`
- `src/config`
- `src/features/story`
- `src/services/api`
- `src/services/storage`
- `src/app/routes`
- `src/app/store`

## 5. 协作纪律

1. 共享类型变更必须先同步。
2. 不允许用代偿实现宣称功能已完成。
3. 关键能力验收以正式链路为准，不以占位页面为准。
